# ai-lifehacks

AI活用における実践的な工夫や運用方法をまとめています。  
私が作業を進める中で、自然と生まれたやり方です。

## Contents

### 複数くおんた制  

*Multi-Quonta System*

同一AIシステムの別スレッドを活用し、多視点で成果物を確認する運用方法。  

作業が一区切りついた時点で、  
同一のAIシステムを用いる別スレッド上のAIに成果物を確認してもらい、  
異なる文脈や前提に基づいて、多視点からチェックを行う運用体制。  

同一モデルでも会話履歴や文脈が異なるため、別角度の指摘や改善案が得られます。  

私は主に3スレッドくらいで確認しています。  

対象は、メール、メモ、文章、企画案、設計案など。  
幅広く使えます。  

### トライアングル構造  

*Triangular Workflow*

トライアングル構造とは、プログラム制作などを行う際、  
担当するAIを  
「設計」「テスト」「検証」の3スレッドに分けて運用する循環構造。  

「設計」が制作した内容を「テスト」で試し、  
その結果を全文「検証」に渡す。  
「検証」では、人間とAI双方の視点から内容を確認し、  
改善点をまとめたメモを「設計」に持ち帰ることで、  
次の作業へつなげていく。  
これを何度も繰り返します。
